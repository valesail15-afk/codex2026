# 项目防反复修改约束（执行版）

更新时间：2026-04-17

## 1. 根因分类（本项目已确认）
- 编码污染：中文字符串被终端或脚本错误处理，出现不可读字符。
- 模式串线：单场/二串一/HG 共用实现过深，改一处影响另一处。
- 口径分散：同一计算口径在多个组件重复实现，导致列表与弹窗不一致。
- 展示分叉：列表与详情不是同一份结构化数据源，出现“内外不一致”。

## 2. 强制约束
- C1 编码约束：任何改动不得引入乱码、占位字符或损坏文本。
- C2 模式隔离：HG 详情必须独立实现，不允许污染单场或二串一。
- C3 单一口径：HG 中奖金额口径统一。
  - 标准盘：`本金 * 赔率`
  - 让球盘：`本金 * (1 + 赔率)`
- C4 数据一致：推荐列表与“查看方案”弹窗必须来源于同一策略结构。
- C5 发布门禁：提交前必须通过文本完整性检查与 TypeScript 校验。

## 3. 执行命令（每次改动后）
```bash
node tools/check-text-integrity.mjs . --deep
node tools/check-staged-text-integrity.mjs
npm run -s lint
```

## 4. Gate 失败处理
- 编码问题：先修文本，再继续业务改动。
- 模式串线：拆分组件边界，不允许在同组件堆叠高风险分支。
- 口径不一致：删除重复实现，统一到单一函数。

## 5. 当前机械守卫
- `tools/check-text-integrity.mjs`
  - 核心/深度文本污染检查
- `tools/check-staged-text-integrity.mjs`
  - 仅检查本次提交新增行，阻断新增乱码与术语漂移
