# 项目防反复修改约束（执行版）

更新时间：2026-04-04

## 1. 根因分类（本项目已确认）
- 编码污染：中文字符串被终端/脚本错误编码，出现 `U+FFFD`、`???`、Mojibake。
- 多模式耦合：`单场/二串一/HG` 共用大组件，修一个模式容易污染另一个模式。
- 计算口径分散：同一公式在多个组件重复实现，导致列表与弹窗不一致。
- 展示规则割裂：列表和详情不使用同一数据源，出现“内外不一致”。

## 2. 强制约束
- C1 编码约束：任何改动不得引入乱码、`???` 占位。
- C2 模式隔离：HG 详情必须独立组件，不得与单场详情复用高风险渲染分支。
- C3 单一口径：HG 中奖金额必须统一口径  
  - 标准盘：`本金 * 赔率`
  - 让球盘：`本金 * (1 + 赔率)`
- C4 数据一致：推荐列表与查看方案弹窗必须来自同一策略结构（同下注项、同赔率、同高亮）。
- C5 发布前守门：提交前必须通过治理脚本。

## 3. 执行命令（每次改动后）
```bash
node tools/governance-gate.mjs
```

## 4. Gate 失败处理
- 出现编码问题：先修编码，再继续业务改动。
- 出现模式污染：拆分组件，不允许继续在同组件堆条件分支。
- 出现口径不一致：删除重复公式，统一调用单一函数/单一实现位置。

## 5. 本次新增的机械守门
- 文件：`tools/governance-gate.mjs`
- 检查项：
  - 文本完整性深度扫描
  - TypeScript 静态检查
  - HG 弹窗组件隔离检查
  - HG 基准注/对冲注规则检查
  - HG 中奖金额口径检查
  - 高频乱码模式扫描
