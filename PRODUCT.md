# 产品文档 - AFK 套利系统（PRODUCT）

> 最后更新：2026-04-22 18:05  
> 适用版本：v1.8.64

---

## 1. 产品目标
AFK 套利系统用于聚合竞彩与皇冠赔率，输出可执行的套利方案，并在前端提供清晰的下注矩阵、收益覆盖和风险提示。系统覆盖三类模式：

- **单场**：竞彩 + 皇冠（标准盘/让球盘）
- **二串一**：竞彩二串一 + 皇冠补单（分两阶段）
- **HG 对冲**：仅使用皇冠胜平负 + 皇冠让球

---

## 2. 架构与模块

### 2.1 前端
- 技术：React + TypeScript + Vite + Ant Design
- 关键页面：
  - 主控面板：`/src/pages/Dashboard.tsx`
  - 比赛列表：`/src/pages/MatchList.tsx`
  - 系统设置：`/src/pages/Settings.tsx`
  - HGA 映射：`/src/pages/HgaTeamAliasSettings.tsx`
- 关键弹窗组件：
  - 单场：`/src/components/SinglePlanDetailContent.tsx`
  - 二串一：`/src/components/ParlayPlanDetailContent.tsx`
  - HG：`/src/components/GoalHedgePlanDetailContent.tsx`

### 2.2 后端
- 技术：Node.js + Express + SQLite
- 关键模块：
  - `src/server/crawler.ts`：抓取与同步
  - `src/server/arbitrageEngine.ts`：套利计算核心
  - `server.ts`：路由、鉴权、配置落库

### 2.3 数据流
1. 抓取器同步竞彩/皇冠赔率并写库。
2. 套利引擎按模式扫描机会并写入机会表。
3. 主控页面按筛选条件读取机会列表。
4. 详情弹窗按机会 ID 拉取明细并实时计算展示。

## 3. 业务规则

### 3.1 单场规则
- 使用竞彩 + 皇冠赔率组合计算。
- 目标是在主胜/平/客胜三种赛果下得到正收益或满足最低利润阈值。
- 支持让球盘映射与高亮。

### 3.2 二串一规则
- 第一场需先满足单场可套利条件。
- 第二场开赛时间晚于第一场 5 小时以上。
- 第一场命中后才触发第二场皇冠补单。
- 支持第二场皇冠赔率临时校准（仅当前弹窗会话生效，不落库）。

### 3.3 HG 对冲规则
- 仅使用皇冠胜平负与皇冠让球赔率。
- 展示层分为：皇冠标准盘 + 皇冠让球盘。
- 中奖计算口径：`中奖金额 = 下注金额 × (1 + 赔率)`（赔率为纯收益赔率）。
- 利润计算口径：`利润 = 中奖金额 + 返水收益 - 总投入`。

## 4. 展示规范

### 4.1 共用矩阵视觉语义
- **橙底白字**：实际下注项
- **淡绿底**：中奖覆盖区域
- **金额颜色**：
  - 下注金额：黑色
  - 实投金额：蓝色
  - 中奖金额：黑色
  - 利润金额：蓝色

### 4.2 主控列表
- 单场、二串一、HG 三类数据互相独立，不共享列表状态。
- 利润率统一红色显示。
- 操作按钮统一蓝底白字，文案固定为“查看方案”。

### 4.3 弹窗交互
- 打开弹窗时显示加载状态。
- 数据加载失败显示错误提示并允许重试。
- 单场/二串一/HG 各自独立组装字段，但共享视觉规范。

## 5. 参数与配置

### 5.1 套利参数
- 竞彩默认返水（%）
- 皇冠默认返水（%）
- 竞彩占比（%）
- 皇冠占比（%）

> 套利参数对有权限用户可见并可保存，保存后触发重算。

### 5.2 HGA 配置
- 启用开关、登录账号、登录密码
- 一键测试登录状态
- 防锁保护：出现“密码错误次数过多”时停止继续尝试抓取

### 5.3 HGA 球队别名映射
- 默认两列：Trade500 / HGA
- 支持新增别名列
- 支持导入导出（管理员）
- 支持自动加入阈值与建议处理

## 7. 质量基线

### 7.1 文案与编码
- 所有页面与文档采用 UTF-8。
- 统一术语：胜/平/负、中奖、利润、利润率、实投、查看方案。
- 乱码检查纳入文本守卫脚本。

### 7.2 响应式
- 桌面端优先单行展示。
- 窄屏允许横向滚动并保证首列可读。
- 列宽有最小值约束，避免内容挤压错位。

### 7.3 回归验证
- 每次改动后至少验证：
  - 抓取流程
  - 三类套利计算
  - 列表与弹窗一致性
