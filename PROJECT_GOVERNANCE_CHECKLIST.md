# 项目治理清单

## 目的
本清单用于记录“问题 -> 根因 -> 约束 -> 验证”的闭环，避免同类问题反复出现。

## 治理项

### G001
- 类型：Bug 修复回灌
- 标题：二串一扫描不得截断比赛池
- 触发原因：二串一引擎可计算出正收益方案，但推荐列表为空。排查发现 `findParlayOpportunities` 曾按固定数量截断比赛，导致后续候选比赛被漏算。
- 约束/修复动作：二串一扫描必须使用完整比赛池；如需性能优化，必须基于明确规则裁剪，不允许硬编码前 N 场。
- 验证方式：重扫后检查 `/api/arbitrage/parlay-opportunities?base_type=jingcai`，确认可返回后续场次组合的真实方案。
- 日期：2026-04-03

### G002
- 类型：Bug 修复回灌
- 标题：二串一策略名禁止使用占位字符
- 触发原因：`best_strategy.name` 曾写入占位串，导致前端显示乱码。
- 约束/修复动作：策略名必须由稳定中文标签生成（如“二串一LP(让胜x让负)”），禁止占位符与编码不明字符。
- 验证方式：重扫后核对接口 `best_strategy.name` 与页面展示一致。
- 日期：2026-04-03

### G003
- 类型：Bug 修复回灌
- 标题：二串一矩阵标签与高亮映射必须使用标准中文
- 触发原因：接口层标签与高亮键曾使用损坏字面量，导致详情页和列表展示不一致。
- 约束/修复动作：统一使用标准标签函数（主胜/平/客胜），禁止在接口层写入损坏字符。
- 验证方式：重扫后检查 `match_1_matrix/match_2_matrix` 标签、列表高亮和详情高亮一致。
- 日期：2026-04-03

### G004
- 类型：Bug 修复回灌
- 标题：Parlay 详情组件不得保留损坏文案
- 触发原因：组件内残留损坏常量，导致“查看方案”出现乱码和说明缺失。
- 约束/修复动作：平台名、下注说明、表头、收益指标和说明文案统一用稳定中文常量。
- 验证方式：静态扫描通过，页面刷新后文案无乱码。
- 日期：2026-04-03

### G005
- 类型：Bug 修复回灌
- 标题：二串一详情禁止跨基准平台用同一 ID 取数
- 触发原因：详情页曾用同一记录 ID 请求不同 `base_type`，触发 404 与内容错位。
- 约束/修复动作：详情页只请求当前基准平台记录；跨平台必须有可映射稳定标识。
- 验证方式：打开详情不再出现跨平台 404，且内容与列表一致。
- 日期：2026-04-03

### G006
- 类型：Bug 修复回灌
- 标题：二串一弹窗必须展示完整下注清单
- 触发原因：仅靠矩阵高亮，用户难以还原完整下注结构。
- 约束/修复动作：弹窗顶部必须明确展示竞彩二串一、第一场皇冠、第二场皇冠的下注项/赔率/金额/实投/说明。
- 验证方式：打开任意二串一方案时，顶部可直接看到完整清单。
- 日期：2026-04-03

### G007
- 类型：Bug 修复回灌
- 标题：二串一详情矩阵必须完整回填赔率
- 触发原因：详情矩阵曾只显示高亮格，其余赔率错误显示为 `-`。
- 约束/修复动作：矩阵必须完整回填标准盘与让球盘可用赔率；高亮只影响样式不影响数据显示。
- 验证方式：详情中竞彩/皇冠胜平负与让球赔率完整显示，仅无数据格允许显示 `-`。
- 日期：2026-04-03

## 执行要求
- 每次出现同类问题，必须新增治理项并补“验证方式”。
- 治理项必须可自动化校验，优先落地为脚本守卫。

### G008
- 类型：新增约束
- 标题：进球对冲算法必须保持 base_type 级隔离
- 触发原因：新增 `goal_hedge` 后，若复用旧筛选/渲染路径，容易污染 `single/hg/parlay` 结果与口径。
- 约束/修复动作：`goal_hedge` 必须独立扫描、独立查询、独立主控渲染与详情弹窗；仅共享底层赔率源，不共享展示模板与筛选逻辑。
- 验证方式：切换主控下拉 `single/hg/parlay/goal_hedge`，确认各自列表与详情均正确且互不串数据。
- 日期：2026-04-17
